---
title: "Quick Start"
description: "Get Typeflux recording and inserting text in under five minutes"
---

This guide walks you through building Typeflux from source, launching it as a macOS app, granting the required permissions, and making your first voice input.

## Prerequisites

- macOS 13 or later
- Xcode with Swift 5.9+ toolchain installed (or the Swift toolchain from swift.org)
- A microphone (built-in or external)

## Build and launch

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/mylxsw/typeflux.git
    cd typeflux
    ```
  </Step>
  <Step title="Build and launch the app">
    ```bash
    make run
    ```
    This compiles the Swift package, assembles a `.app` bundle at `~/Applications/Typeflux Dev.app`, and launches it. Typeflux appears in your menu bar as a small icon.

    <Tip>
      Use `make dev` instead to keep terminal logs attached. This is useful if you want to see transcription output and debug messages in real time.
    </Tip>
  </Step>
  <Step title="Grant permissions on first launch">
    Typeflux walks you through the required macOS permissions on first launch. You need to grant:

    1. **Microphone** — to record your voice
    2. **Accessibility** — to insert text into other apps
    3. **Speech Recognition** — only if you plan to use Apple Speech

    The app opens the relevant System Settings pane for each one. After granting, return to Typeflux and click Continue.

    See [Permissions](/permissions) for details on what each permission is used for.
  </Step>
  <Step title="Choose a speech provider">
    For the easiest getting-started experience, use **Apple Speech** — it requires no API key or download and works on any Mac.

    1. Click the Typeflux icon in the menu bar
    2. Click **Settings**
    3. Go to **Speech Recognition**
    4. Select **Apple Speech** from the provider list
    5. Grant the macOS Speech Recognition permission when prompted

    Alternatively, you can configure **Whisper API** with an OpenAI key or download a **Local Model** for higher accuracy. See [Speech-to-Text Providers](/providers/speech-to-text) for all options.
  </Step>
  <Step title="Make your first voice input">
    1. Open any text editor, browser text field, or document
    2. Click inside a text area to focus it
    3. Press and hold the activation hotkey (default: depends on your configuration — check **Settings → Hotkeys**)
    4. Speak a sentence
    5. Release the hotkey

    The floating overlay appears while you speak, then disappears when transcription is complete. Your spoken text is inserted at the cursor.

    <Note>
      If text appears in the clipboard but not in the text field, Accessibility permission may need to be re-granted. See [Permissions](/permissions) for troubleshooting.
    </Note>
  </Step>
</Steps>
